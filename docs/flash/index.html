esp32-c5

TRIDENT — C5 Satellite

POSEIDON's wireless drop-node. 5 GHz WiFi scan / deauth / PMKID + 802.15.4 Zigbee sniffer. ESP-NOW link to the Cardputer mothership.

Heads-up — the one-click button may not work yet. esp-web-tools 10.2.1 (the newest release) still bundles esptool-js 0.5.7, which has a confirmed ESP32-C5 flash-detection bug: it reaches the port picker, then fails with "Flash ID: 0 / Failed to initialize." The fix shipped in esptool-js 0.6.0 but no esp-web-tools build has bundled it yet (issue #687). Try the button first — if it errors, use the browser flasher below.
✅ Recommended: Espressif Web Flasher (browser, no install)
  1. Download the 3 firmware files from bin/trident-c5/: bootloader.bin, partition-table.bin, poseidon_c5_node.bin.
  2. Open espressif.github.io/esptool-js in Chrome or Edge (this tool is already on esptool-js 0.6.x, so the C5 works).
  3. Plug in TRIDENT, click Connect, pick its serial port.
  4. Add three flash rows and set each Flash Address + file: 
    0x2000   bootloader.bin
0x8000   partition-table.bin
0x10000  poseidon_c5_node.bin
  5. Leave flash mode dio, size 4MB (or Keep), then click Program. Done — unplug and re-plug to boot.
Alternative: command line (if you have Python esptool) 
pip install esptool
esptool --chip esp32c5 -p COM<n> --baud 460800 write_flash \
  --flash_mode dio --flash_freq 80m --flash_size 4MB \
  0x2000   bootloader.bin \
  0x8000   partition-table.bin \
  0x10000  poseidon_c5_node.bin
(On Linux/macOS the port looks like /dev/ttyACM0 or /dev/ttyUSB0.)
